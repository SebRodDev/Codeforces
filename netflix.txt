Question 1:
Question (not word for word, general concept tho):
Imagine that you are trying to process revenue from several countries, you're given the following arrays:
S, where S[i] represents the number of people that reside in some country i.
R, where R[i] represents the subscription that some person in country i pays.
T , where T[i]  represents the total time in seconds that it would take to process the revenue made from country i.
Now, processing these requests take time, and all of these requests are passed through a single server. This server, however, isn't the greatest server, and can actually fail.
This server actually doesn't fail so long as you are processing less than 4 countries.
However, for more than 4 countries, there's concerns about this server failing and can even fail WHILE processing revenue but there's a couple of constraints.
For some value c, the server is guaranteed to NOT fail in the middle of processing c requests.
After c requests, the server can fail at some uniformly random time. Should this be the case, it's still possible to partially process a revenue request for some country.
Given these constraints, we want to return an array arr  that represents the ordering of the countries we should process for revenue.
If we have less than 4 countries to process, we should order these countries such that we keep the mean wait time to a minimum.
Should we have 4 or more countries to process, we should order these such that we maximize revenue earnings, that way in the event of a server failure/shutdown, we would have processed as much money as possible given the ordering.

Example:
S = [10, 5, 4, 2, 2]
R = [ 6, 5, 9, 5, 2]
T = [12, 5, 6, 2, 4]
c = 2

// Output:
[1,3,2,4,5] (note everything is 1-indexed)
Reasoning: Since total number of countries = 5, we maximize revenue instead of wait time. For c=2 if we process countries 1 & 3 first, we get 10*6 + 4*9 = $96. After this, shutdown can happen while processing a countries revenue, thus if we process 2,4,5 in that respective order, it can be shown that an arbitrary server shutdown at any time t while processing in this order maximizes revenue in comparison to shutdown at t with any other ordering.
